# 🏚️ 🍌 Lumière - AI 智能家装规划师 (AI Home Renovation Planner)

这是一个基于多智能体（Multi-Agent）系统构建的高级智能家装平台。它能够分析您房间的照片，为您制定个性化的翻新计划，并利用底层强大的多模态 AI 能力为您生成照片级真实的室内设计渲染图。

## ✨ 核心特色与架构重构

本项目基于核心智能体逻辑进行了**彻底的前后端分离重构**，前端为 `roomGPT_frontend`（Next.js）：

*   **🎨 全新前端 (Next.js + roomGPT)**:
   *   采用 roomGPT 前端交互体验，实现面向中文用户的文案。
   *   前端直接对接本地 FastAPI 接口（`/api/chat-with-image`）。
*   **⚙️ 纯净 API 后端 (Python + FastAPI)**:
    *   将 ADK Agent 逻辑通过 FastAPI 封装为标准的 RESTful API 接口。
    *   无缝处理前端的文字交互、多模态图片上传及流式/批量事件响应。
    *   支持本地自动代理渲染图文件静态映射服务。

## 功能特性

- **🔍 智能图像分析**: 上传房间现状照片和心仪的灵感参考图 —— 智能体将自动检测并综合分析它们
- **🎨 照片级逼真渲染**: 使用 大模型 为您翻新后的空间生成具有专业水准的精美渲染图
- **💰 预算感知规划**: 根据您的预算限制量身定制装修建议
- **📊 完整的装修路线图**: 提供时间进度表、详细的预算明细、承包商清单以及具体的行动执行清单
- **🤖 多智能体协同编排**: 演示了“协调者(Coordinator)/分发者(Dispatcher) + 顺序流水线(Sequential Pipeline)”的智能体设计模式
- **✏️ 持续迭代优化**: 根据您的反馈可随时编辑、调整生成的渲染图

## 工作原理

系统采用 **协调者/分发者模式 (Coordinator/Dispatcher)**，包含三个各司其职的专业智能体：

1. **视觉评估师 (Visual Assessor)** 📸
   - 分析上传的房间照片（布局、现状、大致尺寸等）
   - 从参考图片中提取设计风格
   - 评估改造成本，发现改进空间

2. **设计规划师 (Design Planner)** 🎨
   - 制定符合预算的设计方案
   - 给出具体的材料、颜色和灯具设备的建议
   - 优先考虑能大幅提升质感的改造项目

3. **项目协调员 (Project Coordinator)** 🏗️
   - 生成全面的房屋翻新行动路线图
   - 绘制翻新后空间的逼真渲染图
   - 提供各类成本细目、时间表和实操步骤

## 快速开始

当前架构经过了前后端分离改造。

1. **启动后端服务 (FastAPI)**
```bash
# 安装基础依赖
pip install -r requirements.txt

# 启动服务 (默认运行在 8000 端口)
python -m uvicorn server:app --host 0.0.0.0 --port 8000
```

### 2. 启动前端服务 (Next.js)
```bash
# 进入前端目录
cd roomGPT_frontend

# 安装依赖 (推荐使用国内镜像加速)
npm install --registry=https://registry.npmmirror.com

# 启动开发服务器
npm run dev
```

### 3. 开始体验
在浏览器中打开前端项目地址（通常为 `http://localhost:3000`）开始体验全新的智能家装规划师。

### 本地数据说明
- 会话、消息和图片元数据会保存在本地 SQLite：`.adk/planner.db`
- ADK 生成和上传的图片 artifact 会保存在 `.adk/artifacts`
- 这些文件仅用于本地开发运行，不会自动上传到远程服务

## 工具与能力支持

- **google_search (谷歌搜索)**: 全网检索装修成本、建材价格和最新设计趋势
- **estimate_renovation_cost (装修成本估算)**: 依据房屋空间类型、面积范围和改造深度计算装修开销
- **calculate_timeline (建设工期计算)**: 预估工程的耗时周期
- **generate_renovation_rendering (生成装修渲染)**: 创作极其逼真的照片级渲染图
- **edit_renovation_rendering (编辑装修渲染)**: 根据细节反馈重新渲染或局部修改图形
- **版本化数字资产记录 (Versioned artifacts)**: 系统会自动追踪并记录每一版生成过的渲染图

## 多智能体模式架构

本项目生动演示了 **协调者/分发者 + 顺序流水线 (Coordinator/Dispatcher + Sequential Pipeline)** 模式：

```
协调员 (Root Agent - 根节点智能体)
    ├── 信息智能体 (Info Agent - 负责快速问答)
    └── 规划流水线 (Planning Pipeline - 顺序执行)
          ├── 视觉评估师 (Visual Assessor - 图像分析)
          ├── 设计规划师 (Design Planner - 明确各类规格细则)
          └── 项目协调员 (Project Coordinator - 产生渲染图 + 行动路线图)
```

**为什么采用这种模式？**
- **高效**: 只在需要时才触发更长链的工作流
- **模块化**: 每个智能体各司其职，责任边界清晰
- **可扩展**: 添加新功能或引入新角色更为方便
- **生产可用**: 这是当前企业级应用中最成熟、最具实战意义的智能体系统架构模式
